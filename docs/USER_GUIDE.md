## 1. 安装与启动

### 1.1 环境要求

| 依赖 | 版本 |
| --- | --- |
| Python | 3.10+ |
| PySide6 | ≥ 6.6, < 7 |
| PySide6-Fluent-Widgets | ≥ 1.6, < 2.0 |
| pylink-square | **1.6.0**（不要装 2.x，breaking API change） |
| J-Link 驱动 | SEGGER 官方最新版（Flasher PRO / J-Link 系列） |

### 1.2 启动

双击 `start.bat`（已激活 venv 并启动 `src/main.py`）。

首次启动会在 `%APPDATA%\JLinkRTTViewer\` 创建 `user_prefs.json` 和 `logs/`。

### 1.3 Nuitka 打包

```
build_nuitka.bat
```

会输出独立 `J-LinkRTTViewer.exe`，无需 Python 环境运行。

## 2. RTT 监控页

主要功能页，最常用。

### 2.1 控制栏（顶部）

| 控件 | 用途 |
| --- | --- |
| 目标设备 | MCU 型号下拉，可输入关键字（如 "STM32H750"）自动补全匹配 |
| 接口 | SWD / JTAG |
| 速度 (kHz) | 1000 / 4000 / 8000 / 12000 等常用速度 |
| RTT 通道 | 默认 0；如果 MCU 端用 `SEGGER_RTT_printf(N, ...)` 发送，需匹配 |
| ▶ 连接 / ⏸ 断开 | 主要动作按钮，未连接显示 ▶（播放）图标，已连接显示 ⏸（暂停）图标 |
| 🔄 重置目标 | 通过 J-Link 软复位 MCU |

**连接成功**后：按钮文字变成"断开"+ ⏸ 图标；设备信息卡片标题自动追加目标芯片摘要（如 "设备信息 — STM32H750VB / SWD / 8000 kHz"），不展开也能看到当前连接的芯片。

### 2.2 设备信息卡片

默认折叠。点右上 ⌄ 按钮展开，显示：
- J-Link 固件版本 / 硬件版本 / 序列号
- 核心名称 / Core ID / CPU 类型
- 当前目标设备 / 接口 / 速度

### 2.3 选项栏

| 控件 | 说明 |
| --- | --- |
| 自动滚动 | 收到新数据自动滚动到底部；用户手动滚到中间时**不会**被拉回（4 像素余量判断） |
| 暂停接收 | 暂停 RTT 数据接收（worker 仍连接，只是不读 buffer） |
| 电源输出 | J-Link 5V 输出开关（仅连接后可用） |
| 实时日志记录 | 勾选后所有 RTT 数据同步写到 `%APPDATA%\JLinkRTTViewer\logs\rtt_yyyymmdd_HHMMSS.log` |
| 会话标记 | 输入文字后点"插入标记"，在显示区追加 `──── 文本 ────` 高亮分隔行 |
| A− / A+ | RTT 显示字号 ±1（8-32 范围，持久化） |
| 清除 | 清空显示区（不影响日志文件） |
| 💾 保存当前 | 把当前显示内容另存为 .log 文件 |

### 2.4 显示区

- 支持 ANSI 转义颜色（标准 8 色 + bright 8 色，覆盖 30-37/40-47/90-97/100-107）
- 长行自动按窗口宽度换行
- 默认显示最近 10000 行（设置页可调 1k-100k）
- 暗黑/明亮主题自动响应

### 2.5 搜索栏

- 输入文字，按 ↑/↓ 在 buffer 内查找上一个/下一个
- 显示"当前命中 / 总匹配数"（如 `3/12`），200ms 节流避免大日志按键卡顿
- 找不到时回卷一次

### 2.6 发送栏

- 文本模式：直接输入 UTF-8 字符串发送（**可下拉历史**：发送框是 EditableComboBox，最近 50 条命令直接选用）
- Hex 模式：勾选 Hex 后输入 `AA BB CC` 或 `AABBCC`，奇数长度自动末位补 0
- 发送后自动清空输入框（避免 Enter+Enter 误重发）
- 发送历史保存最近 50 条（持久化）

### 2.7 搜索高亮全部匹配

搜索栏输入关键字回车后，**所有匹配位置背景色叠浅黄**，方便一眼定位所有出现处（上限 500 处避免大日志卡顿）。配合 ↑/↓ 在匹配之间跳转。

### 2.8 底部状态栏

页面最下方常驻 4 段状态，所有数据 1 秒刷新一次：

| 段 | 显示 |
| --- | --- |
| ● 连接状态 | 灰色"未连接" / 绿色"已连接 <芯片型号>" |
| 速率 | `B/s · 行/s` 即时速率（最近 1 秒内 RTT 接收字节数和行数） |
| 累计 | `总字节 · 总行数 · 会话时长(HH:MM:SS)`，按当前连接会话计算，断开后重置 |
| 编码 | 当前 RTT 解码编码（utf-8 / gbk / utf-16-le / latin-1 / ascii） |

观察 MCU 输出速率是否符合预期、判断是否丢包，看这里就行。

## 3. 内存查看页

> **页面支持垂直滚动**：当窗口被任务栏挤窄或屏幕较小时，所有卡片（读取 / Hex 显示 / 类型解析 / 导出固件 / 写内存）整体可上下滚动，hex 显示区不会被压扁。

读取 MCU 内存地址范围并以 hex dump 展示，支持数据类型解析、地址跳转、Hex 搜索、自动刷新、固件导出。

### 3.1 读取卡片

**第一行**：
- 起始地址（支持 `0x...` 或十进制）
- 大小（最大 16 MB）
- 字节/行：8 / 16 / 32 切换
- 读取 / 清空

**第二行**：
- 自动刷新（已连接时启用）+ 周期 1-60s
- **高亮变化**：默认开启，连续两次读取同地址同长度时，**变化的字节背景标红**——便于观察 RAM/寄存器实时变化
- 跳转到地址（输入 `0x...`，定位到该字节并选中）
- Hex 搜索（输入 `AA BB CC` 在已读取缓冲内 find_next，带回卷）
- A− / A+ 调整 hex 字号（字号独立于 RTT 字号）

### 3.2 Hex Dump 显示区

格式：`0xAAAAAAAA:  HH HH HH HH  HH HH HH HH ... |ascii|`

**鼠标悬停在任意字节字符上** → 弹出 tooltip 显示：
```
地址 0x08000010  (+16)
u32 LE: 305419896 (0x12345678)
u32 BE: 2018915346 (0x78563412)
u16 LE: 22136 (0x5678)
```

### 3.3 右侧数据类型解析面板

随光标在 hex 区移动**实时**更新：
- 光标地址（绝对地址 + 缓冲内 offset）
- 字节序：小端 (LE) / 大端 (BE)
- 类型行：u8 / i8 / u16 / i16 / u32 / i32 / float / double

### 3.4 复制 / 保存

| 按钮 | 作用 |
| --- | --- |
| 复制选中为 Hex 字串 | `12 34 56 78`（空格分隔，大写）；未选区时复制全部 |
| 复制选中为 ASCII | 不可打印字符显示为 `.` |
| 复制全部为 C 数组 | `uint8_t data[N] = { 0x12, 0x34, ... };` 直接粘贴到代码 |
| 保存全部为 .bin | 二进制原始字节，文件名 `mem_0xADDR_NBYTES.bin` |

### 3.5 固件导出（底部卡片）

按 4 KB 分块流式读取并写入文件，**期间 RTT 接收暂停**：
- 预设：128 KB / 256 KB / 512 KB / 1 MB / 2 MB / 自定义
- 进度条实时更新

### 3.6 写内存（⚠ 高风险）

页面底部"写内存 ⚠"卡片可以直接向 MCU 内存写入字节，常见用途：

- 调试时临时改写一个 SRAM 变量观察行为
- 给某个外设寄存器写入控制字
- 验证某段 RAM 是否可写

**用法**：
1. 输入目标地址（如 `0x20000000`，STM32 SRAM 起始）
2. 输入 Hex 数据（如 `AA BB CC DD`，空格可选）
3. 点 **"写入…"** → 弹出确认对话框，显示完整地址、前 16 字节预览、红字警告
4. 点"确认写入" → 写入；点"取消" → 放弃

**风险**：
- 写错地址（如 flash 控制器寄存器、未映射区域）可能导致 MCU 失去响应直到下次复位
- **不要**写 flash 控制器、PMU、看门狗等敏感外设
- 写入按 32-bit 字进行，末尾不足 4 字节用 `0xFF` 补齐

**安全地址参考**：
- STM32 SRAM：`0x20000000` 起
- 普通 GPIO ODR 寄存器、定时器 ARR 等"业务"外设——MCU 不会因此挂死

## 4. 设置页

### 4.1 外观

| 项 | 说明 |
| --- | --- |
| 主题模式 | 跟随系统 / 浅色 / 深色 |
| 主题色 | 取色对话框，影响按钮/选中等强调色 |
| RTT 显示字体 | 字体下拉（系统字体列表，编程/中文字体置顶）+ 字号 SpinBox。输入"yahei"自动补全到"Microsoft YaHei" |
| UI 界面字体 | 影响侧边栏/按钮/标签等所有 fluent 控件；"恢复默认"回到 Segoe UI 9pt |

### 4.2 RTT 行为

| 项 | 说明 |
| --- | --- |
| 显示区最大行数 | 1000-100000（超出 cap 后旧行自动 drop） |
| RTT 轮询间隔 | 5ms-1s，影响读 buffer 频率。默认 100ms 是经过实测稳定的值 |
| RTT 解码编码 | utf-8（默认）/ gbk / utf-16-le / latin-1 / ascii。切换后立即重建 decoder，无需重连 |
| 日志保存目录 | 默认 `%APPDATA%\JLinkRTTViewer\logs`；可改到自定义路径 |

## 6. 快捷操作 / 提示

### 6.1 配置持久化

所有用户偏好自动保存到 `%APPDATA%\JLinkRTTViewer\user_prefs.json`：
- 窗口几何（位置/大小，下次启动恢复）
- 目标设备 / 接口 / 速度 / RTT 通道
- 自动滚动 / Hex 发送模式
- 主题 / 字体 / 字号 / 编码 / 轮询间隔 / 最大行数 / 日志目录
- 发送历史

写盘有 200ms 节流（拖动窗口/调字号不会每帧刷盘），关闭窗口时强制 flush。

### 6.2 状态切换

| 操作 | 即时反馈 |
| --- | --- |
| 点击"连接"（▶） | 按钮立即变"连接中…"灰色 |
| 点击"断开"（⏸） | 按钮**立即**变回"连接"（▶）（不等 worker 回信号，乐观恢复 UI） |
| 连接成功 | 按钮变 "断开"+⏸，状态栏"● 已连接 <芯片>"变绿，自动激活 重置/发送/电源 等控件 |
| 连接失败 | InfoBar 弹出错误，按钮自动恢复 |
| 收到 worker error | log_message → InfoBar 弹出；如按钮卡"连接中…"也强制恢复 |

### 6.3 字号一致性

- RTT 监控页 A−/A+ 改 `font_size`（影响 RTT 显示）
- 内存查看页 A−/A+ 改 `memory_font_size`（hex dump 独立字号）
- 设置页 SpinBox 是同一份配置，三处实时同步

### 6.4 重置

如果偏好文件损坏或想"重置出厂"：
1. 退出程序
2. 删除 `%APPDATA%\JLinkRTTViewer\user_prefs.json`
3. 重新启动

## 7. 常见问题

### Q1: 连接成功但 RTT 显示区一直空白

**最常见原因**：UI 的 RTT 通道与 MCU 端 `SEGGER_RTT_printf(N, "...")` 通道不一致。

- 默认 MCU 在通道 0 发送，UI 默认也是 0
- `user_prefs.json` 保存了上次选择 — 切到其他通道后会一直记住
- **快速验证**：把通道调到 0 试试

### Q2: 中文乱码

设置页 → RTT 解码编码 → 试试 `gbk`（中国大陆 MCU 工程常见）或 `utf-16-le`。

### Q3: pylink "no connected JLink" 警告

pylink 1.6.0 内部状态机的正常现象（断开时残留），不影响功能。日志会记录但 UI 不弹错。

### Q4: J-Link 连接慢 / 第一次连接超时

pylink 1.6.0 的 `connect(target)` 在 STM32 大型芯片上实测最长 ~3s。**断开** `close()` 实测同样最长 3s。

### Q5: 窗口底部被任务栏遮挡 / 内存页卡片被挤压看不清

- 应用启动时已自动把窗口夹回 `availableGeometry()`（任务栏之外的可用区域）
- 内存查看页整页内容支持**垂直滚动**——窗口压扁时滚动条出现，不会再压扁 hex 显示区
- 如仍遇到 RTT 页底部状态栏被遮挡，请向上拖窗口或缩小高度后再调整位置

### Q6: 数据接收很快时 UI 卡顿

- RTT 读循环跑在独立 native 线程，主线程 50ms 节流批量渲染
- 高速场景下可调高"显示区最大行数"上限，或勾"暂停接收"临时停止刷新

### Q7: 想看 J-Link 序列号、固件版本

连接成功后，**设备信息**卡片（控制栏下方）点 ⌄ 展开。

### Q8: 一次性查看大范围内存（如 1 MB）

- 用"内存查看"页直接读：最大 16 MB
- 但内存读会一次性占用 worker 线程，期间 RTT 接收暂停
- 长时间观察某段 RAM 变化，建议用"自动刷新"+ 较小范围（如 128 字节）+ 开"高亮变化"

## 附：项目结构

```
src/
├── core/
│   ├── ansi_parser.py     # ANSI 转义解析
│   ├── config_service.py  # 配置 + 节流落盘
│   ├── jlink_worker.py    # pylink 后台 worker（QObject + 外部 QThread）
│   ├── logger.py          # 日志
│   └── memory_service.py  # 内存读 / hex dump / 类型解析 / C 数组
├── ui/
│   ├── _infobar.py        # InfoBar 工具函数
│   ├── about_page.py
│   ├── main_window.py     # FluentWindow + 导航
│   ├── memory_viewer_page.py
│   ├── rtt_monitor_page.py
│   └── settings_page.py
└── main.py

tests/                     # pytest，67 个测试
CLAUDE.md                  # 维护踩坑笔记
```

CLAUDE.md 包含项目维护时的全部踩坑经验（线程模型、PySide6 跨线程信号、pylink 1.6.0 连接顺序等），有兴趣的开发者可参考。
